# User Profile For Explanations

Keep this file minimal. Notion is the durable user model.

## Preference Index

Primary Notion page:

- `User Preferences` in the Tools DB: `https://www.notion.so/34ded17cd10981f5a1edf4db91be6a3d`

Plugin users should replace this URL with their own `User Preferences` page after recreating the database documented in `../../../docs/tools-db-dependency.md`.

Use that page as the high-level preference repository and as a map to lower-level preference material elsewhere in Notion.

## Notion Retrieval

For important explanations, search Notion for:

- the target concept;
- adjacent concepts the user may already understand;
- examples of how the user has written or reasoned about similar ideas;
- the `User Preferences` page if goal or style is uncertain.

Hotkey anchors that often matter:

- agent architecture;
- software systems;
- async/concurrency;
- scaling laws;
- knowledge systems;
- explanation methodology.

## Notion Updates

When the user gives durable feedback, update Notion when appropriate. Prefer page content over metadata.

For non-database page body updates, mark AI-authored content with:

`AI: [AI-authored text]`

If inserting AI-authored fragments into an existing user paragraph, wrap each independent AI insertion in square brackets.

Do not use database entries marked `Creator = AI` as evidence of the user's writing voice. They may still be used as operational preference records.
